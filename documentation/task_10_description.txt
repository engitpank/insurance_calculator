Разница между Git Rebase и Merge?
> Merge объединяет две ветки, создавая merge commit:
Завершена работа над функцией feature/login, хотим слить изменения с веткой main
> Rebase 'перезаписывает' коммиты так, будто они были созданы поверх последней версии целевой ветки. Используется только для локальных коммитов. Пример:
Работаем над функцией в ветке feature/profile, но обнаружили, что в main появились обновления (например, исправления багов).
Мы хотим интегрировать эти изменения в свою ветку, чтобы продолжить разработку без конфликтов.

Использовал rebase для перезапиши изменений в локальном коммите STEP 007 - DONE (4b05313e): нужно было изменить строку в файле .gitignore:
- git rebase -i 4b05313e^ - делает rebase до указанного коммита включительно (^)
- в открывшемся списке для коммита заменяется pick на edit. Файл сохраняется
- Далее добавляю изменения git add .
- Делаю коммит git commit --amend
- Завершаю: git rebase --continue
