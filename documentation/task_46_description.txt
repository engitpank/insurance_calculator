**Что было сделано и как? (краткое описание)**
Предполагаю, что тестовые сценарии должны задавать требования к входным и возвращаемым данным.
POST запрос на эндпоинт /insurance/travel/
0. Должен принимать JSON.
1. Должен возвращать JSON.
2. При переданных валидных данных должен быть возвращен объект Response, содержащий переданные поля и agreementPrice.
3. При переданных валидных данных поле errors не должно быть возвращено.
4. При некорректных данных должно быть возвращено только поле errors, содержащее массив ошибок.
5. При некорректных данных должно быть возвращено массив errors, содержащий поля field и message.
6. Запрос должен быть отклонён, если формат даты не соответствует шаблону "yyyy-MM-dd".
7. Контроллер должен вернуть Response, содержащий дату в формате "yyyy-MM-dd".
